# 图层组合（Group）与锁定（Lock）功能实现计划

## 1. 数据结构升级
- **文件**: `src/ai-image-editor/types/layer.ts`
- **内容**:
  - 扩展 `LayerType` 增加 `'group'`。
  - 扩展 `CanvasLayer` 接口，增加：
    - `parentId: string | null`
    - `children: string[]`
    - `isGroup: boolean`
    - `isLocked: boolean` (替代或同步现有的 `locked`)

## 2. 组合（Group）功能实现
- **文件**: `src/ai-image-editor/canvas/services/group.ts` (新增)
- **功能**:
  - `groupSelected(manager)`:
    - 获取当前选中的对象（如果是 `ActiveSelection`）。
    - 检查其中是否有被锁定的对象（若有则不允许组合）。
    - 创建 `fabric.Group`，将选中对象添加进去。
    - **关键**: 处理 CanvasLayer 的更新。
      - 创建一个新的 Layer 代表 Group。
      - 更新原 Layer 的 `parentId` 指向新 Group ID。
      - 更新 Group Layer 的 `children` 列表。
      - **注意**: Fabric.Group 会接管渲染，原对象实际上在 Canvas 的 `_objects` 列表中被移除，转而存在于 Group 的 `_objects` 中。我们需要确保 `LayerManager` 中的 `layers` 列表依然能反映这种层级，或者调整 `layers` 列表只展示顶层节点（取决于 UI 需求，通常图层列表展示树状结构或扁平结构）。鉴于目前是扁平列表，我们可能需要保持扁平，但通过 `parentId` 标记层级。
      - **修正**: Fabric 的 `Group` 机制下，子对象不再直接位于 canvas 的 objects 列表中。这意味着 `manager.layers` 中的 `fabricObject` 引用依然有效，但 `canvas.getObjects()` 不再返回它们。我们需要调整 `LayerManager` 的同步逻辑，或者明确 `CanvasLayer` 代表的是逻辑图层。
  - `ungroupSelected(manager)`:
    - 获取当前选中的 Group。
    - 调用 `group.toActiveSelection()` (Fabric API) 还原。
    - 更新 Layer 数据：移除 Group Layer，清空子节点的 `parentId`，清空 Group Layer 的 `children`。

## 3. 锁定（Lock）功能增强
- **文件**: `src/ai-image-editor/canvas/services/modify.ts`, `src/ai-image-editor/canvas/services/interactive.ts`
- **功能**:
  - 更新 `toggleLock`：
    - 设置 `isLocked`。
    - 递归处理：如果是 Group，需同时锁定/解锁其所有子节点（或者 Fabric Group 锁定本身就足够？通常锁定 Group 即可，但为了数据一致性，建议标记子节点）。
    - 增强 `setInteractive`：不仅设置 `selectable`, `evented`，还要处理 `hasControls` 等。
  - 交互限制：
    - 在选择逻辑中（`useFabricCanvas`），确保框选时跳过锁定对象（Fabric 的 `selectionKey` 配合 `selectable: false` 通常能自动处理，但需验证）。

## 4. UI 与快捷键集成
- **文件**: `src/ai-image-editor/hooks/useHotkeys.ts`
  - 绑定 `Mod+G` (Group), `Mod+Shift+G` (Ungroup), `Mod+L` (Lock)。

## 5. 序列化支持
- **文件**: `src/ai-image-editor/canvas/services/serialize.ts`

  - `loadFromJSON` 时需重建 Group 结构（这通常比较复杂，因为 Fabric 的 `loadFromJSON` 会自动重建 `fabric.Group`，我们需要重新关联 Layer ID）。

**关于 Fabric Group 的特别说明**:
Fabric 的 `Group` 对象在创建时，会改变子对象的坐标系（从绝对坐标变为相对于 Group 中心的坐标）。在 `ungroup` 时又会变回来。这意味着 `CanvasLayer` 中的 `fabricObject` 引用虽然不变，但其属性（`left`, `top`）会发生剧烈变化。这通常由 Fabric 内部处理，我们只需要确保 Layer 数据结构（ID 关系）正确即可。

**注意**: 
用户要求 "Group 节点本身不渲染内容"。Fabric 的 Group 默认是透明容器，符合要求。
用户要求 "多选框选默认不包含锁定节点"。这需要覆盖 Fabric 的默认框选行为，或者确保锁定节点的 `evented` 为 `false`。
