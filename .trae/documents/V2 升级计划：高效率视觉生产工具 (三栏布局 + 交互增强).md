# V2 AI 图片编辑器升级计划 - 高效率视觉生产工具

## 1. 核心架构与状态升级
- **交互模式变更**：从 V1 的“静态展示”升级为“完全交互”。
  - 修改 `useFabricCanvas` 初始化配置，开启 `selection: true`。
  - 修改 `LayerManager`，默认对象属性改为 `selectable: true, evented: true`。
- **状态同步机制**：建立 Fabric Canvas 到 React State 的单向数据流。
  - 监听 Fabric 事件 (`object:added`, `object:removed`, `object:modified`, `selection:updated`)。
  - 实时更新 React 中的 `layers` 列表和 `activeObject` 状态，用于驱动左侧图层列表和右侧属性面板。

## 3. 详细实施步骤

### 阶段一：底层能力升级 (Canvas & LayerManager)
1.  **扩展图层模型**：在 `types/layer.ts` 中增加 `text` 类型，以及 `visible`, `locked`, `zIndex` 字段。
2.  **增强 LayerManager**：
    -   支持 `addText(text, options)`。
    -   实现图层排序方法 (`bringForward`, `sendBackwards`)。
    -   实现 `lock/unlock`, `show/hide`。
    -   实现 `updateLayer` 方法，用于响应右侧属性面板的修改。
3.  **Canvas 交互启用**：
    -   更新 `useFabricCanvas`，允许多选、框选。
    -   添加 Canvas 事件监听器，将画布变更同步回 React `useReducer` 或 `useState`。

### 阶段二：界面布局与左侧操作区
1.  **三栏布局实现**：使用 Flex/Grid 实现 左(300px) - 中(自适应) - 右(280px) 布局。
2.  **左侧功能开发**：
    -   **素材模块**：复用并优化 `ImportImageButton`，支持连续导入。
    -   **文本模块**：添加“添加标题”、“添加副标题”按钮。
    -   **图层列表**：渲染图层列表，支持点击选中、显隐切换、锁定切换、拖拽排序（或上下移按钮）。

### 阶段三：右侧属性面板与画布编辑
1.  **属性面板开发**：
    -   基于当前选中的 Fabric 对象类型（Image/Text/Group）渲染不同组件。
    -   **通用**：图层名称、删除、复制、层级调整。
    -   **图片**：透明度滑块、尺寸信息。
    -   **文本**：内容输入框、字号滑块、颜色选择器、对齐按钮、行高。
2.  **画布交互增强**：
    -   集成**辅助对齐线**（使用 Fabric 的 `object:moving` 事件计算吸附）。
    -   实现**尺寸切换响应**：切换 Canvas 比例时，尝试保持图层相对位置或居中适配。

### 阶段四：导出与优化
1.  **导出升级**：
    -   导出前自动清除选区（`discardActiveObject`）。
    -   自动隐藏辅助线（如果有渲染辅助线对象）。
    -   文件命名：`daydo_poster_{timestamp}.png`。
2.  **体验打磨**：
    -   添加快捷键支持（Delete 删除，Ctrl+C/V 复制粘贴）。
    -   确保 Undo/Redo 的基础（预留，V2 暂不深入，但架构要支持）。

## 5. 变更风险与应对
-   **状态不同步**：Fabric 是命令式，React 是声明式。
    -   *应对*：以 Fabric 为单一数据源（Source of Truth），React 仅作为渲染层，通过事件触发 `forceUpdate` 或同步关键状态。
-   **性能问题**：图层过多时 React 渲染频繁。
    -   *应对*：使用 `useMemo` 和 `React.memo` 优化图层列表和属性面板；Canvas 事件做防抖处理。
