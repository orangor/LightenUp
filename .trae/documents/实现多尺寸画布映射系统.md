# 多尺寸画布映射 (Variant Canvas) 实施计划

## 1. 数据结构扩展
- **目标**: 扩展 `LayerManager` 以支持多画布（Master + Variants）。
- **设计**:
    -   `LayerManager` 不再只管理单一画布状态，而是作为“画布管理器”的入口。但为了最小化重构，我们引入一个新的上层管理器 `ProjectManager` 或者在 `LayerManager` 内部增加多画布切换逻辑。
    -   考虑到现有架构 `LayerManager` 强绑定了一个 `fabric.Canvas` 实例。我们采用“单 Canvas 实例，多数据状态切换”的策略。
    -   新增 `CanvasVariant` 接口:
        ```typescript
        interface CanvasVariant {
            id: string
            name: string
            role: 'master' | 'variant'
            width: number
            height: number
            layers: any[] // 序列化后的图层数据
            safeArea: SafeAreaConfig
            sourceCanvasId?: string // 如果是 variant，指向 master id
        }
        ```
    -   在 `LayerManager` 中添加 `variants` 列表和 `currentVariantId`。

## 2. 核心功能实现
### 2.1 状态管理
-   在 `LayerManager` 中：
    -   `variants: CanvasVariant[]`
    -   `activeVariantId: string`
    -   `switchVariant(id: string)`: 保存当前状态 -> 加载目标状态 -> 调整 Canvas 尺寸 -> 渲染。
    -   `addVariant(config: { width, height, name })`: 基于当前 Master 生成新 Variant。

### 2.2 智能映射算法 (Master -> Variant)
-   当创建 Variant 时：
    -   计算 Master 到 Variant 的缩放比例 `scale = min(vW/mW, vH/mH)`。
    -   遍历 Master 图层，克隆并调整：
        -   `left`, `top`: 按比例缩放并居中（Cover/Contain 策略）。
        -   `scaleX`, `scaleY`: 乘以 `scale`。
        -   `fontSize`: 乘以 `scale`。
    -   安全区映射：按比例调整 `top`, `bottom`, `left`, `right`。

### 2.3 UI 交互
-   **CanvasToolbar (中间顶部)**:
    -   新增“画布列表”下拉菜单。
    -   显示当前画布名称和尺寸。
    -   “新建衍生画布”按钮。
-   **新建弹窗**:
    -   输入名称。
    -   选择尺寸预设（1:1, 9:16, 16:9 等）。

## 4. 关键算法 (Mapping)
-   **Scale Factor**: `s = Math.min(targetW / sourceW, targetH / sourceH)`
-   **Centering**:
    -   `offsetX = (targetW - sourceW * s) / 2`
    -   `offsetY = (targetH - sourceH * s) / 2`
-   **Object Transform**:
    -   `obj.left = obj.left * s + offsetX`
    -   `obj.top = obj.top * s + offsetY`
    -   `obj.scaleX *= s`, `obj.scaleY *= s`
